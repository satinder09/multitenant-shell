# Users API Reference

Complete technical documentation for all Users API endpoints.


## GET /users

**Get all users**

Retrieves a paginated list of users in the current tenant with optional filtering and sorting capabilities.


### Parameters

| Name | Type | In | Required | Description |
|------|------|----|---------|-----------| 
| page | integer | query | ❌ | Page number for pagination (starts from 1) |
| limit | integer | query | ❌ | Number of users per page (maximum 100) |
| search | string | query | ❌ | Search term to filter users by name or email |
| role | string | query | ❌ | Filter users by role |
| status | string | query | ❌ | Filter users by status |
| sortBy | string | query | ❌ | Field to sort by |
| sortOrder | string | query | ❌ | Sort order |


### Responses


#### 200 - Users retrieved successfully

**Schema:**

```json
{
  "type": "object",
  "properties": {
    "data": {
      "type": "array",
      "items": {
        "$ref": "#/components/schemas/User"
      }
    },
    "pagination": {
      "$ref": "#/components/schemas/PaginationInfo"
    },
    "filters": {
      "$ref": "#/components/schemas/FilterInfo"
    }
  }
}
```


#### 401 - No description


#### 403 - No description


### Security

This endpoint requires authentication:
- Authentication
- JWT-auth
