# Users API

The Users API provides comprehensive user management functionality within tenant contexts.

## Overview

The Users API allows you to:

- **Create new users** with complete profile information
- **List all users** with advanced filtering and pagination
- **Get user details** by ID
- **Update user information** including roles and status
- **Delete users** when necessary

## Base URL

All API requests should be made to:
```
http://localhost:4000/api/v1/users
```

## Authentication

All endpoints require authentication using JWT tokens:

```bash
Authorization: Bearer <your-jwt-token>
```

## Endpoints

### Create User

Create a new user in the current tenant.

**POST** `/users`

**Request Body:**
```json
{
  "firstName": "John",
  "lastName": "Doe",
  "email": "john.doe@example.com",
  "role": "user",
  "status": "active"
}
```

**Response:**
```json
{
  "success": true,
  "data": {
    "id": "user-123",
    "firstName": "John",
    "lastName": "Doe",
    "email": "john.doe@example.com",
    "role": "user",
    "status": "active",
    "createdAt": "2024-01-01T12:00:00Z"
  }
}
```

### List Users

Get a paginated list of users with optional filtering.

**GET** `/users`

**Query Parameters:**
- `page` (optional): Page number (default: 1)
- `limit` (optional): Items per page (default: 10, max: 100)
- `search` (optional): Search by name or email
- `role` (optional): Filter by role (`user`, `admin`, `manager`)
- `status` (optional): Filter by status (`active`, `inactive`, `suspended`)

**Response:**
```json
{
  "success": true,
  "data": [
    {
      "id": "user-123",
      "firstName": "John",
      "lastName": "Doe",
      "email": "john.doe@example.com",
      "role": "user",
      "status": "active"
    }
  ],
  "pagination": {
    "page": 1,
    "limit": 10,
    "total": 1,
    "pages": 1
  }
}
```

### Get User

Get details of a specific user by ID.

**GET** `/users/:id`

**Response:**
```json
{
  "success": true,
  "data": {
    "id": "user-123",
    "firstName": "John",
    "lastName": "Doe",
    "email": "john.doe@example.com",
    "role": "user",
    "status": "active",
    "createdAt": "2024-01-01T12:00:00Z",
    "updatedAt": "2024-01-01T12:00:00Z"
  }
}
```

### Update User

Update user information.

**PATCH** `/users/:id`

**Request Body:**
```json
{
  "firstName": "Jane",
  "role": "admin",
  "status": "active"
}
```

**Response:**
```json
{
  "success": true,
  "data": {
    "id": "user-123",
    "firstName": "Jane",
    "lastName": "Doe",
    "email": "john.doe@example.com",
    "role": "admin",
    "status": "active",
    "updatedAt": "2024-01-01T13:00:00Z"
  }
}
```

### Delete User

Delete a user by ID.

**DELETE** `/users/:id`

**Response:**
```json
{
  "success": true,
  "message": "User deleted successfully"
}
```

## Error Responses

All endpoints return consistent error responses:

```json
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Invalid request data",
    "details": {
      "email": "Invalid email format"
    }
  }
}
```

## Status Codes

- **200**: Success
- **201**: Created
- **400**: Bad Request
- **401**: Unauthorized
- **403**: Forbidden
- **404**: Not Found
- **422**: Validation Error
- **500**: Internal Server Error
